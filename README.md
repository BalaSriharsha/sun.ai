# SunnyAI - Agentic Platform

SunnyAI is an open-source, powerful Generative AI Agentic Platform. It provides a unified workspace to build, manage, and interact with Large Language Models, autonomous Agents, semantic Knowledge Bases, Tools, MCP (Model Context Protocol) Servers, and complex Workflows.

![Playground UI Interface](docs/screenshot.png) <!-- Replace with an actual screenshot path if available -->

## Features

- **Multi-Modal Chat**: Chat natively with leading models (OpenAI, Anthropic, Gemini, Azure, etc.) with support for both text and file uploads (Images, PDFs, Docs, Audio, Video).
- **Autonomous Agents**: Create robust AI agents equipped with custom tools. The agents leverage a fully-managed tool-calling loop to perform complex tasks.
- **Knowledge Bases (RAG)**: Create and manage Knowledge Bases. Upload documents (PDFs, Word, CSV, Audio, Video) which are instantly parsed, embedded, and made fully searchable.
- **MCP Servers Integration**: Register Model Context Protocol (MCP) servers to instantly extend your agents with external capabilities like GitHub, Google Drive, or Slack integrations.
- **Visual Workflows**: A node-based UI (using React Flow) to construct intricate agentic workflows, connecting inputs, models, tools, and outputs dynamically.
- **Observability**: Built-in latency tracking, token usage, and step-by-step LLM step visualizers.
- **Multi-Workspace**: Cleanly isolate environments using Workspaces and Organizations.

## Architecture & Tech Stack

- **Frontend**: [Next.js](https://nextjs.org) (App Router), React, React Flow, Clerk (Authentication), Vanilla CSS / Tailwind.
- **Backend**: [FastAPI](https://fastapi.tiangolo.com) (Python), Langchain/Langgraph, LiteLLM (for multi-provider routing), Uvicorn.
- **Database**: PostgreSQL (with pgvector for embeddings). Includes fallback support for SQLite.
- **Document Parsing**: Tesseract OCR, ffmpeg, PyPDF2, and Pandas.

## Getting Started: Self-Hosting Guide

The easiest way to self-host and run SunnyAI is via Docker Compose. This ensures all system dependencies (like Tesseract and ffmpeg for parsing documents) and the vector database are flawlessly configured.

### Prerequisites

- [Docker](https://docs.docker.com/get-docker/) & [Docker Compose](https://docs.docker.com/compose/install/) installed.

### 1. Clone the Repository

```bash
git clone https://github.com/BalaSriharsha/SunnyAI.git
cd SunnyAI
```

### 2. Configure Environment Variables

You need to provide API keys for authentication and any LLM providers you plan to use. 

**Backend (`backend/.env`)**
Create the file and add:
```env
# Required for sending emails (if applicable)
RESEND_API_KEY=your_resend_api_key

# Database Connection (Leave as is for Docker Compose pgvector setup)
CURRENT_DATABASE=postgres
DATABASE_URL=postgresql://postgres:password@db:5432/agentic_platform

# Add any default LLM keys here, though you can also add them via the UI Provider settings
OPENAI_API_KEY=your_openai_api_key
ANTHROPIC_API_KEY=your_anthropic_key
```

**Frontend (`frontend/.env.local`)**
Create the file and add your Clerk authentication keys:
```env
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_...
CLERK_SECRET_KEY=sk_test_...
NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in
NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up
NEXT_PUBLIC_CLERK_SIGN_IN_FALLBACK_REDIRECT_URL=/
NEXT_PUBLIC_CLERK_SIGN_UP_FALLBACK_REDIRECT_URL=/

# The backend URL the browser should talk to.
# Leave as localhost:8000 for local development/testing.
NEXT_PUBLIC_API_URL=http://localhost:8000/api
```

### 3. Build and Run via Docker Compose

Run the following command from the root of the project:

```bash
docker-compose up -d --build
```

This spins up three containers:
1. `db`: The PostgreSQL + pgvector database.
2. `backend`: The FastAPI Python backend (accessible on port `8000`).
3. `frontend`: The Next.js web application (accessible on port `3000`).

### 4. Access the Platform

- Open your browser and navigate to **[http://localhost:3000](http://localhost:3000)**.
- If you need to hit the backend API directly, the autogenerated Swagger docs are available at **[http://localhost:8000/docs](http://localhost:8000/docs)**.

### 5. Stopping the Platform

To stop the containers while preserving your database volumes and file attachments:

```bash
docker-compose down
```

## Local Development Setup (Without Docker)

If you wish to run the servers locally for active development:

### Backend
1. Ensure Python 3.11+, ffmpeg, and tesseract are installed on your OS.
2. `cd backend`
3. `python -m venv venv && source venv/bin/activate`
4. `pip install -r requirements.txt`
5. Ensure a PostgreSQL DB is running, or switch `CURRENT_DATABASE=sqlite` in `.env`.
6. `uvicorn main:app --reload --port 8000`

### Frontend
1. Ensure Node.js 18+ is installed.
2. `cd frontend`
3. `npm install`
4. `npm run dev`
5. App will be served at [http://localhost:3000](http://localhost:3000).
